# LnxTerm 구현 계획

ST-Link V3 Mini를 통한 임베디드 장치 디버그/로그 수집용 시리얼 터미널 GUI 프로그램.

## 현재 릴리스 범위
- 버전: **v1.8.1**
- 목표: 기존 기능 안정화 + 문자열 통계 CSV 정합성 강화 + 문서 체계 정비

## 기술 스택
- Python 3 + PyQt6 + pyserial + python-dotenv
- PyInstaller 단독 실행 파일 빌드
- GitHub CLI(`gh`)를 통한 릴리스 관리

## 핵심 기능

| 기능 | 현재 상태 |
|------|------|
| 시리얼 통신 | 포트 스캔, 연결/해제, QThread 수신, LF 기반 라인 파싱 |
| 자동 재연결 | 끊김 3초 자동 재연결 + 로그 재개 |
| 포트 점유 감지 | Lockfile + `fuser`/`lsof` + `TIOCEXCL` 검증 |
| 다중 인스턴스 | 싱글톤 제한 없이 다중 실행 허용 |
| 터미널 출력 | `[YYYY-MM-DD HH:MM:SS.mmm]` 타임스탬프, RX/TX 컬러 구분 |
| 로그 관리 | `.env` `LOG_DIR` 기반 저장, 로그 경로/경로 복사 UX |
| 문자열 찾기 | 최대 10개 문자열 등록, 매칭 누적 카운트/시작 시각 |
| 문자열 통계 | 통계 파일 생성/복사, 항목 일괄 초기화, CSV 스키마 고정 |
| 레이아웃 | 사이드바-터미널 분할 리사이즈(QSplitter) |
| 검색 | Ctrl+F + 하이라이트 + 탐색 |
| 명령 입력 | 하단 입력바, Enter/히스토리(↑↓) |
| 배포 | 빌드 스크립트 + GitHub Release 산출물 제공 |

## 작업 항목

### 완료 항목
1. 사이드바 문자열 통계 CSV 스키마 정합화
   - 저장 열 순서: 문자열/집계시점/누적카운트/대소문자구분/로그
   - 로그 열은 대상 문자열이 포함된 원문 1줄 전체 저장
2. 문서 파일 대문자 정리
   - `doc/IMPLEMENTATION_PLAN.MD`
   - `doc/PROJECT_STATUS.MD`
   - `doc/WALKTHROUGH.MD`
   - `doc/PROMPT.MD`
3. 배포 준비
   - `./build_exe.sh` 수행 완료
   - 릴리스 버전 반영 및 릴리스 등록

### 진행 중/잔여 항목
1. 통계 CSV 샘플 고정/문서화: 스키마 예시를 문서에 추가 정리
2. 통계 파일 경로/버그 조건에서 동시 접근 안정성 점검
3. UI 테마/아이콘 대비 추가 튜닝(권장)
