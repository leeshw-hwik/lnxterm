# LnxTerm 개발 워크스루

## 프로젝트 개요
ST-Link V3 Mini를 이용한 임베디드 장치 디버그/로그 수집용 시리얼 터미널 GUI 프로그램.

## 주요 성과
- **GitHub Release**: [LnxTerm v1.7](https://github.com/leeshw-hwik/lnxterm/releases/tag/v1.7) 배포 완료
- **포트 점유 감지 (v1.7)**: Lockfile 확인 및 배타적 접근(`exclusive=True`) 모드 적용으로 안정성 강화
- **다중 인스턴스**: 싱글톤 잠금 없이 여러 창 동시 실행 지원
- **단독 배포본**: Python 없이 실행 가능한 Linux 64-bit 바이너리

## 주요 기능 상세

### 포트 점유 감지 (v1.7)
단순 프로세스 확인을 넘어 리눅스 시스템 레벨의 점유 상태를 3단계로 확인합니다.
1. **Lockfile 확인**: `/var/lock/LCK..tty*` 파일을 확인하여 표준 잠금 상태를 감지합니다.
2. **프로세스 확인**: `fuser` 및 `lsof` 명령으로 해당 디바이스 파일을 열고 있는 프로세스를 찾습니다.
3. **배타적 접근(TIOCEXCL)**: 연결 시 `exclusive=True` 옵션을 사용하여 커널 레벨에서 다른 프로세스의 접근을 차단합니다. 다른 프로그램이 이 모드로 열려 있다면 연결 시도 시 즉시 에러(Busy)가 발생합니다.

### 다중 인스턴스
- `./lnxterm &`으로 여러 창을 띄워 각각 다른 포트를 모니터링할 수 있습니다.
- 동일한 포트에 접근하려고 하면 위에서 설명한 점유 감지 로직에 의해 경고창이 뜨며 연결이 차단될 수 있습니다.
