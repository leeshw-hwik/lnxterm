# LnxTerm 개발 워크스루

## 프로젝트 개요
ST-Link V3 Mini를 이용한 임베디드 장치 디버그/로그 수집용 시리얼 터미널 GUI 프로그램.

## 주요 성과
- **GitHub Release 완료**: [LnxTerm v1.6](https://github.com/leeshw-hwik/lnxterm/releases/tag/v1.6) 배포 완료
- **단독 배포본 제공**: Python 설치 없이 사용 가능한 Linux 64-bit 전용 바이너리 제공
- **보안 환경 구축**: `.env` 파일 보호 및 깃 클론 시 예제 파일(`.env.example`) 제공
- **안정성 검증**: 비정상 끊김 상황에서의 3초 내 자동 재연결 로직 안정화

## 주요 기능 및 사용법 (최신)

### 1. 가장 쉬운 시작 방법 (GitHub Release)
- [릴리스 페이지](https://github.com/leeshw-hwik/lnxterm/releases)에서 `lnxterm` 파일을 다운로드합니다.
- `chmod +x lnxterm`으로 실행 권한을 주고 바로 실행하면 됩니다.

### 2. 자동 로깅 및 디렉토리 관리
- 프로그램이 처음 실행될 때 로그를 저장할 폴더를 묻습니다.
- 한 번 설정하면 `.env` 파일에 기록되어 다음부터는 자동으로 해당 폴더에 `lnxterm_날짜_시간.log` 파일을 생성합니다.

### 3. 디버깅 최적화 UI
- **색상 구분**: TX 데이터는 파란색, RX 데이터는 초록색으로 표시되어 방향을 직관적으로 알 수 있습니다.
- **날짜 포함 타임스탬프**: 장시간 로그 수집 시 날짜 변화를 확인할 수 있도록 타임스탬프 형식을 확장했습니다.
- **자동 스크롤 및 검색**: 실시간 로그 확인과 과거 로그 검색(`Ctrl+F`)이 동시에 원활하게 작동합니다.

## 빌드 및 배포 시스템
개발자는 `build_exe.sh`를 실행하여 단 1분 만에 최신 배포본을 만들 수 있습니다. 빌드된 파일은 `dist/lnxterm`에 생성됩니다.

## 검증 결과
- ✅ **배포본 무결성**: 다른 리눅스 PC로 복사 후 의존성 설치 없이 정상 구동 확인
- ✅ **GitHub 연동**: `gh` CLI를 이용한 태그 생성 및 자산 업로드 자동화 확인
- ✅ **보안**: `.env` 파일이 실수로 커밋되거나 AI 컨텍스트에 포함되지 않도록 설정 완료
