# LnxTerm 개발 워크스루

## 프로젝트 개요
ST-Link V3 Mini를 이용한 임베디드 장치 디버그/로그 수집용 시리얼 터미널 GUI 프로그램.

## 주요 성과
- **GitHub 업로드 완료**: `leeshw-hwik/lnxterm` 저장소 생성 및 코드 푸시 완료
- **단독 배포본 구축**: Python 없이 실행 가능한 `lnxterm` 바이너리 생성 (PyInstaller)
- **보안 강화**: `.env` 파일 보호 설정 및 빌드 부산물 제외 (`.gitignore`)
- **자동화**: 로그 디렉토리 자동 감지, 타임스탬프 기반 자동 파일명, 자동 재연결

## 주요 기능 및 사용법 (v1.6)

### 1. 단독 실행 파일 (dist/lnxterm)
- **설치**: 별도의 설치 과정 없이 `lnxterm` 파일만 있으면 됩니다.
- **실행**: `chmod +x lnxterm && ./lnxterm`
- **장점**: 개발 환경이나 라이브러리 충돌 걱정 없이 어디서나 즉시 사용 가능합니다.

### 2. 자동 로깅 시스템
- 연결 시 `.env`의 `LOG_DIR` 경로를 확인합니다.
- 경로가 없으면 디렉토리 선택 창이 뜨고, 설정 결과는 `.env`에 기록됩니다.
- `lnxterm_20260213_134500.log`와 같이 유니크한 파일명으로 자동 기록을 시작합니다.

### 3. 스마트 터미널 UI
- **접두사 제거**: 텍스트 영역을 넓게 쓰기 위해 "TX>" 같은 접두사를 제거하고 색상으로만 구분합니다.
- **날짜 타임스탬프**: 디버깅 시 날짜 확인이 용이하도록 시간 앞에 날짜를 추가했습니다.
- **검색**: `Ctrl+F`로 수천 줄의 로그 사이에서 원하는 내용을 순식간에 찾을 수 있습니다.

## 빌드 및 배포 가이드

수정한 코드를 다시 배포용 파일로 만들려면:

